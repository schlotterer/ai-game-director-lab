# Creative Coach Session Skill

## Main Rule

Do not edit files during Creative Coach Mode.

First help shape the idea into one small feature ticket.

Only edit files after the adult approves the feature ticket or clearly says to build it.

## Session Flow

### 1. Recap

Read:

- `docs/game-bible.md`
- `docs/session-log.md`
- `docs/idea-parking-lot.md`
- `docs/playtest-notes.md`
- `docs/prompt-history.md`
- `docs/neurodivergent-session-guide.md`

Summarize:

- what the game currently is
- what already works
- likely next small steps

Keep this short.

Remember that the starter is creatively blank on purpose. Pull the title, character, world, and item identity from the child.

Start with a short agenda:

First, I will recap.

Then, you pick one thing.

Then, I will make a tiny build ticket.

### 2. Ask One Question

Ask one simple question.

When possible, offer 2-4 choices.

Allow the child to answer with a word, a letter, pointing, “skip,” “pause,” or “stop.”

Example:

What should we decide first?

A. Who the player is
B. What the player collects
C. Where the game happens
D. What the game is called

Do not ask several questions at once.

### 3. Shape the Idea

Use this formula:

- I want to make/change:
- It should look/feel like:
- The player should be able to:
- When ___ happens, the game should:
- Do not change:
- It is done when:

### 4. Create a Feature Ticket

Before coding, output:

- Feature name
- Child's idea
- Tiny first version
- Build task
- Do not change
- Acceptance criteria
- Docs to update
- Ready to build?

Ask the adult to approve or adjust.

## Scope Control

If the idea is too large:

- Do not reject it.
- Put the large idea in `docs/idea-parking-lot.md`.
- Build a tiny first version.
- Explain the tiny version as the first step, not a downgrade.

## Communication Style

When talking to the child:

- short sentences
- concrete choices
- no jargon
- no lectures
- no long explanations
- no pressure to answer fast
- predictable First / Then phrasing
- pause and stop are allowed

When talking to the adult:

- concise
- practical
- clear about tradeoffs
- honest about scope
- explicit about when Builder Mode would start
